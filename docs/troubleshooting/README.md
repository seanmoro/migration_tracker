# Troubleshooting Guides

This directory contains guides for debugging, testing, and fixing common issues.

## Categories

### Debug Guides
- `DEBUG_*.md` - Debugging specific issues (API, storage domains, static files, etc.)

### Check Guides
- `CHECK_*.md` - Commands to check system status, databases, schemas, etc.

### Test Guides
- `TEST_*.md` - Testing procedures for various features

### Fix Guides
- `FIX_*.md` - Solutions to specific problems

### Query Guides
- `QUERY_*.md` - Database query examples and procedures

### Other
- `TROUBLESHOOTING_NETWORK.md` - Network troubleshooting
- `backend-troubleshooting.md` - Backend-specific issues
- `DIAGNOSE_*.md` - Diagnostic procedures
- `VERIFY_*.md` - Verification steps
- `KILL_AND_RESTART.md` - Restart procedures
- `SET_POSTGRES_PASSWORD.md` - Password configuration
- `FORCE_REBUILD_FRONTEND.md` - Frontend rebuild procedures
- `FIX_CACHE_ISSUE.md` - Cache-related fixes
- `QUICK_REBUILD_RESTART.md` - Quick rebuild/restart reference
- `QUICK_TEST_STEPS.md` - Quick testing reference

## Finding Help

1. **Specific Issue**: Look for a file matching your issue (e.g., `DEBUG_500_ERROR.md` for 500 errors)
2. **Database Issues**: Check `CHECK_*.md` and `QUERY_*.md` files
3. **Deployment Issues**: See `FIX_DEPLOYMENT_*.md` files
4. **General**: Start with `backend-troubleshooting.md` or `TROUBLESHOOTING_NETWORK.md`
